## Key Terms (Glossary)

| Term | Definition |
|------|------------|
| **Shnarf** | Commitment hash calculated from blob data, used for verification |
| **Conflation** | Process of grouping multiple L2 blocks into a single batch for proving |
| **Batch** | Group of conflated L2 blocks ready for proof generation |
| **Blob** | EIP-4844 data container for submitting compressed L2 data to L1 |
| **Aggregation** | Recursive proof combining multiple execution proofs |
| **Finalization** | L1 transaction that verifies ZK proof and commits L2 state |
| **Rolling hash** | Recursive hash of L2→L1 messages for Merkle tree construction |
| **Traces** | EVM execution matrices generated by tracer, input to prover |

## Data Flow

```
L2 Transactions
      ↓
Sequencer (block production)
      ↓
Tracer (generate execution traces)
      ↓
Coordinator (conflate blocks into batches)
      ↓
Prover (generate ZK proofs: execution → compression → aggregation)
      ↓
Coordinator (submit blobs + finalize on L1)
      ↓
L1 Contract (verify proof, update state)
```

## Cross-Chain Messaging

- **L1 → L2**: Message sent on L1 → Coordinator anchors hash → Claimable on L2
- **L2 → L1**: Message sent on L2 → Included in finalization → Claimable on L1 with Merkle proof

## When Reviewing PRs

Before reviewing changes, check:

1. **Contracts changes**: Read `/documentation/components/contracts.md` for architecture
2. **Coordinator changes**: Read `/documentation/components/coordinator.md` for data flows
3. **Cross-package changes**: Read `/documentation/architecture/OVERVIEW.md` for interactions
