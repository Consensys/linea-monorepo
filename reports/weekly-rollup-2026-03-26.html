Weekly Rollup Competitive Report

Reporting window: 2026-03-19 to 2026-03-26 UTC

Method note: This report includes only developments whose effective date falls within the 7-day reporting window and only from the allowlisted official source registry. A second pass explicitly checked official blogs, docs/notices, and forums in addition to GitHub; no additional qualifying non-GitHub items were confirmed in-window, so the final report remains GitHub-dominant. Older items may be referenced briefly for context but are not scored as current-week developments.

Executive Summary

This was a baseline week because no prior normalized dataset was available for comparison. After a second pass across official blogs, docs/notices, and governance forums, the strongest confirmed in-window signals still came from official GitHub activity.

Optimism had the most strategically important week in the reviewed source set. It merged both a new ZKDisputeGame into OP Stack dispute infrastructure and activation-time deployment plus liquidity bootstrapping for interop ETH bridge contracts, indicating continued progress on both proof infrastructure and cross-chain execution.

Outside OP Stack, Arbitrum introduced owner-configurable tip collection in ArbOS v60+, Starknet changed internal sequencer-network shard distribution and hashing in Apollo Propeller, and Taiko shipped a mainnet client release preparing operators for the upcoming Shasta fork. zkSync and Base showed smaller but still relevant prover and deployment-path hardening. The broader non-GitHub sweep mostly surfaced out-of-window or weak-signal items rather than additional current-week developments.

Top Ecosystem Themes

  • Proof systems are getting more operationally integrated. Optimism's ZKDisputeGame merge is the clearest example this week, while zkSync's prover release focused on autoscaler resilience and Base continued tightening TEE-prover deployment paths.
  • Interop work is moving from design toward activation logic. Optimism's interop ETH bridge change now deploys and funds bridge-related contracts as part of network activation rather than leaving them as follow-on steps.
  • Operator-facing configuration is becoming a competitive surface. Arbitrum exposed fee-routing behavior as an ArbOS control, and Taiko shipped a release with explicit new Shasta and Pacaya flags ahead of a dated fork.
  • Sequencer and network-internal data paths continue to evolve. Starknet's Apollo Propeller update changed shard packaging, encoding, and Merkle validation in a way that explicitly carries compatibility risk.

Competitor Analysis

Arbitrum

Arbitrum's clearest in-window signal was a merged Nitro PR that adds owner-configurable tip collection for ArbOS v60+ via new ArbOwner / ArbOwnerPublic precompiles: https://github.com/OffchainLabs/nitro/pull/4515

The change does not alter default behavior automatically, but it creates a new runtime control over how transaction tips are handled. That matters because fee-routing policy is becoming more explicit and operator-configurable rather than remaining purely implicit in protocol behavior.

Optimism / OP Stack

Optimism merged a spec-compliant ZKDisputeGame into its dispute infrastructure as game type 10, with permissionless lifecycle handling, DelayedWETH bond custody, and proof-binding to msg.sender: https://github.com/ethereum-optimism/optimism/pull/19606

Separately, Optimism added activation-time deployment and funding for SuperchainETHBridge, ETHLiquidity, and L2ToL2CrossDomainMessenger, including bootstrap liquidity for ETH bridging and matching Go/Rust implementation changes: https://github.com/ethereum-optimism/optimism/pull/19648

Taken together, these are the most consequential signals this week. They show OP Stack progressing simultaneously on proof/dispute machinery and on operationalizing superchain interop.

zkSync

zkSync's strongest reviewed current-week signal was prover release v25.2.0: https://github.com/matter-labs/zksync-era/releases/tag/prover-v25.2.0

The release added a GCP signer, HTTP compression for API and EN sync paths, and a fix for prover-autoscaler resource-exhaustion detection plus aggressive-mode thresholding. This looks like operator and infrastructure hardening, not a major protocol-direction change, but it is still useful evidence of active prover-stack iteration.

Scroll

No material allowlisted in-window item surfaced from the reviewed official Scroll blogs, docs, forums, or priority repositories. That should be treated as a coverage result, not as evidence of inactivity across all possible channels.

Base

Base merged a contracts change that allows TEEProverRegistry.initialize() to take an initialProposer, making proposer registration atomic during deployment and removing a separate registration step from deployment flow: https://github.com/base/contracts/pull/220

This is not a headline-grabbing launch, but it is a concrete signal that Base continues to streamline TEE- and proof-adjacent deployment ergonomics.

Starknet

Starknet merged a high-risk Apollo Propeller PR that changes peer data packaging from a single shard to a ShardsOfPeer collection and changes Merkle leaf hashing to use proto-encoded bytes: https://github.com/starkware-libs/sequencer/pull/12914

The PR explicitly flags backward-compatibility risk if peers still rely on the old encoding. Even though this is an internal-sequencer signal rather than an external product announcement, it is strategically relevant because it points to ongoing sequencing and data-distribution evolution.

Taiko

Taiko published taiko-alethia-client v2.2.0 as a mainnet release for the upcoming Shasta fork, with activation scheduled for 2026-04-02 13:15:00 UTC: https://github.com/taikoxyz/taiko-mono/releases/tag/taiko-alethia-client-v2.2.0

The release adds new Shasta and Pacaya flags, improves Shasta event-sync resilience, and makes finalized-checkpoint handling more tolerant. This is a meaningful operator-readiness milestone because it ties technical changes to a dated mainnet fork window.

What Matters Most For Linea

  • Optimism's ZKDisputeGame work is the most important proof-system signal in the current week because it suggests increasing operational maturity around ZK-assisted dispute resolution inside a widely reused stack.
  • Optimism's interop activation logic is the clearest competitive signal on native cross-chain UX. It is worth tracking whether activation-time deployment plus liquidity bootstrapping becomes a market expectation for interoperable rollups.
  • Arbitrum's fee-policy configurability and Taiko's fork-preparation release both highlight a broader pattern: operator-facing controls and release readiness are becoming product-level differentiators, not just implementation details.
  • Starknet's Apollo Propeller change is a reminder that internal sequencing and distribution improvements may create defensible advantages before they show up in public messaging.
