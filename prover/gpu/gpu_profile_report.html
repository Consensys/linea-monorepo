Global Optimization Recommendations

1. Increase Occupancy: Many kernels show occupancy limited to 6 warps/SM. With 188 SMs on Blackwell, this limits parallelism. Consider reducing register usage or shared memory per block, or using smaller block sizes to allow more concurrent blocks per SM.
2. NTT Kernels Dominate: NTT (DIF/DIT) butterflies account for the majority of kernel time. Each stage launches a separate kernel with moderate DRAM utilization (~40-60%). Fusing multiple stages into a single kernel using shared memory could reduce global memory round-trips.
3. Memory-Bound Elementwise Ops: KB add/mul/sub kernels hit 60-65% of peak DRAM throughput but only ~9% SM. These are memory-bound — performance scales with memory bandwidth. Fusing consecutive elementwise operations (e.g., add+mul, scale+bitrev) would reduce memory traffic.
4. ScaleByPowers is Compute-Heavy: The ScaleByPowers kernel shows 68% SM utilization but only 13% DRAM — it's compute-bound due to chained multiplications (g^i computed via prefix product). This is well-optimized; further improvement would require algorithmic changes.
5. Kernel Launch Overhead: Small kernels (< 10 µs) are dominated by launch overhead. For batch operations, the batch NTT API already amortizes this cost. Consider extending this pattern to other operations (batch add, batch mul).
